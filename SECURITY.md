# Security Policy

## Supported Versions
Only the latest major version is currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| v3.x.x  | :white_check_mark: |
| v2.x.x  | :x:                |
| v1.x.x  | :x:                |

## Reporting a Vulnerability
We take the security of the Aura X Financial Operating System very seriously.

If you believe you have found a security vulnerability in Aura X, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please send an email to security@aurax.io. 

You should receive a response within 24 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

Please include the following details with your report:
* Description of the location and potential impact of the vulnerability.
* A detailed description of the steps required to reproduce the vulnerability (POC scripts, screenshots, and compressed packet captures are all helpful to us).
* Any relevant background information or context.

## Bug Bounty Program
Aura X operates a bug bounty program for critical smart contract and infrastructure vulnerabilities. Eligible reports may receive bounties based on the severity of the issue.

### Severity Matrix
* **Critical**: Remote Code Execution, significant fund loss, key compromise. (Up to $250,000)
* **High**: Privilege escalation, partial fund loss, severe data leak. (Up to $50,000)
* **Medium**: DoS, moderate data exposure. (Up to $10,000)
* **Low**: Information disclosure, minor bugs. (Up to $1,000)

We will coordinate a public disclosure with you once the issue has been resolved.
